What it does

This plugin replaces Zen-Cart's default CSS button functionality with some CSS3 styling, based on the initial output provided by http://www.cssbuttongenerator.com/. The buttons produced have rounded corners in the more recent browsers and "fall back" to squared corners on non-CSS3 aware browsers.

How to use it

There is one core-file overwrite in this mod; you should always backup your cart’s database and files prior to making any changes. Copy the catalog files to your cart, after renaming the "YOUR_TEMPLATE" directory to match your custom template’s name:

  1. /includes/functions/html_output.php
  2. /includes/templates/YOUR_TEMPLATE/css/stylesheet_css_buttons.css

Edit the css file to customize the buttons' background, border and text colors to match your website's scheme.

 Link 

The button text for the built-in Zen-Cart buttons is found in the alt-tag text in /includes/languages/english/YOUR_TEMPLATE/button_names.php. For example, the text for the "Submit" button above comes from this PHP define:

define('BUTTON_SUBMIT_ALT', 'Submit');

Once you have copied the files, log into your admin and set Configuration->Layout Settings->CSS Buttons to Yes.

Note: Zen Cart's standard processing in the function zen_image_submit (also in /includes/functions/html_output.php) forces a default to the button's image if the button's alt-text is more than 30 characters in length. This "trick" only applies to the submit-type buttons; a similar check does not occur for link-type buttons!

Note 2: You can style an individual button by using the button's name as a selector. Remember to place these definitions at the bottom of the stylesheet_css_buttons.css file and make sure that the non-Hover styling is specified before the Hover styling: 

span.normal_button.button_sold_out_sm {
  background-color: blue;
}
span.normal_button.button_sold_out_smHover {
  background-color: red;
}
